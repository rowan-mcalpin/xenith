GamepadEx

class GamepadEx(gamepad: Gamepad, triggerThreshold: Double = 0.6, triggerMoveThreshold: Double = 0.1, joystickMoveThreshold: Double = 0.1, reverseYValues: Boolean = true)

This class is primarily used to carry the references to all of the Buttons, Triggers, and Joysticks of the gamepad. It contains all the references to the various Controls that the gamepad contains.

Parameters

gamepad

the Qualcomm gamepad reference

triggerThreshold

the minimum amount the triggers need to be pressed to be considered "pressed"

triggerMoveThreshold

the minimum amount the triggers need to move to be considered "moving"

joystickMoveThreshold

the minimum amount the joysticks need to move to be considered "moving"

reverseYValues

whether to reverse the y values for the joysticks

class Button(reference: () -> Boolean) : GamepadEx.Control

This class corresponds to a single button on a gamepad. It extends the default Qualcomm functionality by having variables for justPressed and justReleased, as well as commands that can be bound and automatically run when the corresponding condition is met.

class Joystick(referenceX: () -> Float, referenceY: () -> Float, referenceButton: () -> Boolean, reverseYValues: Boolean = true) : GamepadEx.Control

This is the most complicated of the control classes. It has a reference to the x and y values as well as a built-in button and automatic reversing of the y-axis, if desired.

Trigger
Link copied to clipboard
class Trigger(reference: () -> Float) : GamepadEx.Control

This class corresponds to a single trigger (left or right) on a gamepad. It is slightly more complicated than a Button because it has a value between 0 and 1 instead of just true or false. It also includes the justPressed and justReleased variables and their corresponding commands, but also a publicly accessible value variable and a command that is called whenever the trigger moves.
